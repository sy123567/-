## 2. Mock 与真实后端切换

复制环境变量文件：

```bash
cp .env.example .env.local
```

默认独立运行：

```dotenv
VITE_USE_MOCKS=true
VITE_API_BASE=http://localhost:8080
```

连接 Spring Boot：

```dotenv
VITE_USE_MOCKS=false
VITE_API_BASE=http://localhost:8080
```

`src/api/client.ts` 中的 `request<T>()` 会请求 `${VITE_API_BASE}${path}`。前端开发服务器重启后环境变量才会生效。生产环境不要把 JWT secret 放在 Vite 变量中；这里只放后端公开 URL。

当前前端关闭 Mock 后已经直接调用的接口：

```text
GET /api/trips
GET /api/users
GET /api/trips/{id}
GET /api/guides
```

PASS 2 页面目前通过丰富的 Mock adapter 展示完整交互；后端按下方契约补齐后，可将对应 adapter 方法逐步切换为真实请求。路径、字段名和枚举值保持固定。

## 3. 统一响应契约

所有业务响应建议使用：

```json
{
  "success": true,
  "code": "OK",
  "message": "ok",
  "data": {},
  "traceId": "a2f7..."
}
```

分页：

```json
{
  "success": true,
  "code": "OK",
  "message": "ok",
  "data": {
    "items": [],
    "page": 1,
    "pageSize": 20,
    "total": 0
  }
}
```

失败：

```json
{
  "success": false,
  "code": "DATE_CONFLICT",
  "message": "成员可用日期没有交集",
  "data": null,
  "traceId": "a2f7..."
}
```

如果后端决定保留 Spring Boot 常见的直接 DTO 返回形式，前端 `request<T>()` 需要统一去掉 `data` 包装；推荐从项目开始就采用上述 `Result<T>`，避免每个页面单独兼容。

采用 `Result<T>` 时，真实 API 分支应将当前的 `request<T>()` 改成统一解包，而不是让页面感知响应信封：

```ts
type Result<T> = {
  success: boolean;
  code: string;
  message: string;
  data: T;
  traceId?: string;
};

async function request<T>(path: string, init?: RequestInit): Promise<T> {
  const response = await fetch(`${base}${path}`, {
    headers: { "Content-Type": "application/json", ...authHeader() },
    ...init,
  });
  const result = (await response.json()) as Result<T>;
  if (!response.ok || !result.success) {
    throw new Error(result.message || `请求失败：${response.status}`);
  }
  return result.data;
}
```

这样 `api.dashboard()` 仍然可以声明 `Promise<DashboardData>`，而后端统一返回 `Result<DashboardData>`。

## 4. 认证、跨域与时间

登录后保存 `token`，真实 API 请求统一增加：

```http
Authorization: Bearer <JWT>
Content-Type: application/json
```

后端 JWT 过期返回 401，前端应清理 token 并导航 `/login`。认证接口：

```text
POST /api/auth/register
POST /api/auth/login
POST /api/auth/logout
GET  /api/users/me
```

Spring Boot 开发环境允许：

```text
http://localhost:5173
http://127.0.0.1:5173
```

生产环境使用明确的前端域名白名单。允许方法 `GET,POST,PUT,PATCH,DELETE,OPTIONS`，允许请求头 `Authorization,Content-Type,Idempotency-Key`。WebSocket `/ws` 也必须允许前端 Origin。

后端统一返回 ISO-8601 时间，例如：

```text
2025-04-19T14:30:00+08:00
```

前端类型中的 `plannedStart`、`plannedEnd`、`startTime`、`endTime` 均按字符串接收；不要返回本地化中文时间。

## 7. 前端当前基础调用与适配建议

当前 `src/api/client.ts` 的真实分支：

```ts
api.dashboard()
  GET /api/trips
  GET /api/users

api.guides()
  GET /api/guides

api.trip(id)
  GET /api/trips/{id}
```

因此后端最小联调顺序是：

1. `GET /api/users` 返回至少一个 `User`；
2. `GET /api/trips` 返回 `Trip[]`；
3. `GET /api/trips/{id}` 返回完整 `Trip`；
4. `GET /api/guides` 返回 `TravelGuide[]`；
5. 再按页面顺序接入 groups、events、risk、plans、votes 等接口。

`Trip` 返回必须至少包含：

```json
{
  "id": 1,
  "title": "上海春日漫游",
  "status": "ONGOING",
  "startDate": "2025-04-19",
  "endDate": "2025-04-20",
  "totalBudget": 2400,
  "spentBudget": 1260,
  "group": {
    "id": 1,
    "name": "周末慢游组",
    "description": "一起把周末过成一张好看的地图",
    "ownerUser": {},
    "memberCount": 4,
    "roomCode": "SH24-7K"
  },
  "itineraryNodes": [],
  "routes": [],
  "riskScore": 68,
  "riskLabel": "需要留意",
  "destination": "上海 · 2天1夜",
  "roomCode": "SH24-7K"
}
```

如果后端 DTO 使用 `owner_user`、`planned_start` 等 snake_case 字段，必须在 Jackson 全局配置或 DTO `@JsonProperty` 中转换为上面的 camelCase；前端不会同时兼容两套字段名。

持久化层可以继续使用 `per_capita_budget`、`rating_count` 等数据库命名，但 Controller response DTO 必须转换为前端字段 `price`、`reviews`、`saves`；不要把数据库列名直接暴露给 React。

## 8. WebSocket 对接

连接：

```text
ws://localhost:8080/ws
```

SockJS/STOMP 订阅：

```text
/topic/trips/{tripId}
```

消息：

```json
{
  "type": "impact-assessed",
  "tripId": 1,
  "payload": {
    "riskScore": 68,
    "affectedNodeIds": [3]
  },
  "occurredAt": "2025-04-19T09:12:00+08:00"
}
```

`type` 使用以下固定值：

```text
event-ingested
impact-assessed
plan-proposed
vote-updated
plan-accepted
```

收到消息后前端刷新对应 Query key：`["trip", tripId]`、`["risk", tripId]`、`["plans", tripId]` 或 `["notifications"]`。

## 9. CORS 与错误处理检查清单

- [ ] `http://localhost:5173` 可访问 `/api/**`。
- [ ] 预检 OPTIONS 返回 200/204，并包含 `Access-Control-Allow-Headers: Authorization,Content-Type,Idempotency-Key`。
- [ ] 401 不返回 HTML，而是统一 JSON。
- [ ] 409 用于日期无交集、房间码过期、投票状态冲突。
- [ ] 422 用于无可行方案、预算无法满足等业务不可行。
- [ ] 所有写接口支持 traceId 日志关联。
- [ ] 所有金额字段 JSON 使用数字或明确约定的字符串，前后端不要混用。
- [ ] 所有分页接口稳定排序，避免滚动加载重复/丢失。
