# 智能旅游平台 · 模块与功能规范清单（完善版 v3）

> 命名规范
> - **领域（Domain）**：`D1~D7`，业务大类，名词短语。
> - **模块（Module）**：`Mxx`，可独立开发交付的功能单元，统一"名词短语 + 管理/服务"式命名。
> - **功能（Feature）**：`Mxx-n`，动宾短语（动词开头），描述一个可测试的能力点。
> - **英文标识**：给出模块英文名，用于代码包名 / 接口前缀 / 表名前缀。
> - **等级**：🟢 基础 CRUD ｜ 🟡 中等逻辑 ｜ 🔴 技术核心。
> - **优先级**：`P0` MVP 必做 ｜ `P1` 落地补强 ｜ `P2` 运营/加分。

## D1 账号管理（Account Management）

### M01 用户账号 (User Account) 🟢 P0
- M01-1 注册和登录并签发 JWT
- M01-2 注销并失效 Token
- M01-3 编辑个人资料与头像

### M02 认证与权限 (Auth & RBAC) 🟡 P0
- M02-1 拦截并校验请求 Token
- M02-2 角色与权限控制（普通用户 / 群主 / 平台管理员）
- M02-3 接口访问限流（防刷）
- M02-4 记录登录设备与登录日志 `P2`

### M03 合规与协议 (Compliance) 🟢 P1
- M03-1 展示并签署用户协议 / 隐私政策
- M03-2 导出个人数据
- M03-3 记录用户授权与同意状态

## D3 出行前规划（Trip Planning）

### M06 成员约束 (Member Constraint) 🟢 P0
- M06-1 录入可用日期
- M06-2 设置预算上限
- M06-3 添加必访地点
- M06-4 设置体力等级 / 饮食需求 / 无障碍需求
- M06-5 汇总小组约束

### M07 初始方案生成 (Initial Planning) 🔴 P0
- M07-1 计算成员可用日期交集
- M07-2 聚合并去重必访地点
- M07-3 按最低成员预算封顶裁剪
- M07-4 排布行程节点与路线
- M07-5 输出方案取舍说明

### M08 行程与节点管理 (Itinerary Management) 🟢 P0
- M08-1 管理行程（增删改查）
- M08-2 管理行程节点（景点 / 餐饮 / 住宿 / 交通）
- M08-3 调整节点顺序
- M08-4 维护节点状态（正常 / 改期 / 替换 / 取消）

### M09 地图路线与导航 (Map & Routing) 🔴 P1
- M09-1 搜索地点 POI 并获取坐标
- M09-2 计算节点间距离 / 时长 / 费用
- M09-3 切换出行方式（步行 / 驾车 / 公共交通）
- M09-4 生成路线可视化数据（polyline）
- M09-5 外部地图 API 失败时降级为直线估算

### M10 预算与费用 (Budget) 🟢 P0
- M10-1 设置总预算
- M10-2 记录费用明细
- M10-3 统计已用 / 剩余预算
- M10-4 按分类汇总费用
- M10-5 超支预警

## D4 行程中监测（In-Trip Monitoring）

### M11 事件接入 (Event Ingestion) 🟡 P0
- M11-1 手动录入外部事件
- M11-2 生成 Mock 事件（天气 / 交通 / 闭馆 / 大型活动）
- M11-3 对接真实第三方 API（限流 / 解析 / 容错）`P1`
- M11-4 按类型 / 严重度 / 日期筛选事件
- M11-5 查询活跃事件

### M12 影响匹配 (Impact Matching) 🔴 P0
- M12-1 计算事件与节点地理距离（Haversine）
- M12-2 判定事件与节点时间窗重叠
- M12-3 关联事件到受影响节点
- M12-4 评估并分级影响
- M12-5 生成影响评估记录

### M13 风险评分 (Risk Scoring) 🔴 P0
- M13-1 采集多因子（严重度 / 受影响比例 / 成本暴露 / 剩余缓冲时间）
- M13-2 归一加权计算 0–100 风险分
- M13-3 划分风险等级与预警
- M13-4 输出风险因子拆解报告

### M14 持续分析 (Continuous Analysis) 🔴 P1
- M14-1 定时（@Scheduled）重跑影响匹配与风险评分
- M14-2 只分析未来（未发生）节点
- M14-3 新事件触发即时增量分析
- M14-4 去重防止重复生成相同分析 / 方案
- M14-5 分析结果变化时推送通知

## D5 应变恢复与决策（Recovery & Decision）

### M15 自动重规划 (Auto Replanning) 🔴 P0
- M15-1 基于影响评估触发重规划
- M15-2 生成三策略方案（最低额外成本 / 最少延误 / 最少改动）
- M15-3 输出节点变更（改期 / 替换 / 删除）
- M15-4 计算增量成本 / 增量延误 / 改动数量

### M16 协同投票 (Collaborative Voting) 🔴 P0
- M16-1 发起方案投票
- M16-2 成员投票（赞成 / 反对 / 弃权，一人一票）
- M16-3 实时展示投票进度
- M16-4 计票并按过半判定
- M16-5 原子应用采纳方案并驳回其余（幂等）

### M17 变更记录 (Change Log) 🟢 P0
- M17-1 记录采纳方案的变更明细
- M17-2 记录额外成本与退款 / 改签截止时间
- M17-3 关联方案与行程查询变更历史

### M18 消息通知 (Notification) 🟡 P0
- M18-1 统一事件总线聚合各模块通知源
- M18-2 WebSocket 实时广播（订阅鉴权）
- M18-3 站内信收发与列表
- M18-4 标记单条 / 全部已读
- M18-5 离线补发
- M18-6 通知偏好设置 `P2`
