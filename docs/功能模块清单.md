# 功能模块清单（模块 / 子功能拆分）

> 行迹应变 —— 多人出行动态规划与中断恢复平台
> 共 6 大分组、20 个模块，每个模块拆分为子功能，并标注难度与对应前端页面 / 后端接口。
> 难度：🟢 简单增删改查 · 🟡 中等 · 🔴 技术难点

---

## A. 账号与社交（组队入口）

### 模块 1｜用户注册 / 登录 / 鉴权 🟢
- 1.1 用户注册（邮箱 + 密码，密码加密存储）
- 1.2 用户登录，签发 JWT
- 1.3 登出（失效 Token）
- 1.4 获取 / 编辑个人资料（昵称、头像、手机号、城市）
- 1.5 鉴权拦截与当前登录用户获取
- 页面：`/login`、`/register`、`/settings`
- 接口：`POST /api/auth/register`、`POST /api/auth/login`、`POST /api/auth/logout`、`GET/PUT /api/users/me`

### 模块 2｜好友系统 🟢
- 2.1 搜索用户
- 2.2 发送好友申请
- 2.3 处理申请（同意 / 拒绝）
- 2.4 好友列表查看
- 2.5 从好友列表邀请进组
- 页面：`/friends`
- 接口：`GET /api/friends`、`GET /api/friends/requests`、`POST /api/friends/requests`、`POST /api/friends/requests/{id}/accept|reject`

### 模块 3｜房间码组队 🟡
- 3.1 创建出行小组，生成唯一房间码
- 3.2 输入房间码加入小组（校验有效期 / 人数上限）
- 3.3 成员列表与角色（群主 / 成员）
- 3.4 移除成员
- 3.5 群主转让
- 页面：`/groups`、`/groups/:id`
- 接口：`POST /api/groups`、`POST /api/groups/join`、`GET /api/groups/{id}`、`GET /api/groups/{id}/members`、`DELETE /api/groups/{id}/members/{memberId}`、`POST /api/groups/{id}/transfer-owner`

### 模块 4｜成员约束管理 🟢
- 4.1 可用日期区间填写
- 4.2 预算上限设置
- 4.3 必访地点增删（标签式）
- 4.4 体力水平选择
- 4.5 饮食需求
- 4.6 无障碍需求
- 页面：`/groups/:id/constraints/:memberId`
- 接口：`GET/PUT /api/groups/{groupId}/members/{memberId}/constraint`

---

## B. 出行前规划

### 模块 5｜出发前共同规划 / 初始方案生成 🔴
- 5.1 读取全体成员约束
- 5.2 计算可用日期交集
- 5.3 必访地点去重聚合
- 5.4 按最低成员预算封顶、超预算裁剪可选节点
- 5.5 排布行程节点与路线，生成初始行程
- 5.6 输出"取舍说明"（为什么保留 / 删除某节点）
- 页面：`/trips/new`（从小组约束生成）
- 接口：`POST /api/groups/{groupId}/plan`

### 模块 6｜行程与节点管理 🟢
- 6.1 行程 CRUD（创建 / 查看 / 列表 / 删除）
- 6.2 节点增删改（景点 / 餐饮 / 住宿 / 交通）
- 6.3 节点排序调整
- 6.4 节点状态（已规划 / 已确认 / 受影响 / 已替换 / 已取消）
- 页面：`/trips`、`/trips/:id`
- 接口：`GET /api/trips`、`GET /api/trips/{id}`、`POST/PUT/DELETE /api/trips/{id}/nodes`

### 模块 7｜地图路线与导航 🔴
- 7.1 节点间路线（距离 / 时长 / 费用）
- 7.2 出行方式切换（步行 / 驾车 / 公共交通）
- 7.3 路线地图可视化
- 7.4 真实路网距离 / 时长（对接地图 API 或图算法）
- 页面：`/routes`、`/trips/:id`（时间线）
- 接口：`GET /api/trips/{id}/routes`、`POST /api/trips/{id}/routes/recalc`

### 模块 8｜预算与费用管理 🟢
- 8.1 总预算 / 已用 / 剩余展示
- 8.2 费用明细记账（增删改）
- 8.3 按类别 / 节点汇总
- 8.4 超支预警
- 页面：`/budget`
- 接口：`GET /api/trips/{id}/budget`、`POST/PUT/DELETE /api/trips/{id}/expenses`

## C. 行程中监测

### 模块 9｜天气 / 交通 / 城市事件接入 🟡
- 9.1 外部事件录入（手动）
- 9.2 Mock Provider 生成模拟事件（暴雨 / 施工 / 闭馆 / 大型活动）
- 9.3 真实第三方 API 对接（限流 / 字段解析 / 容错）
- 9.4 事件列表与筛选（类型 / 严重度 / 时间）
- 9.5 活跃事件查询
- 页面：`/events`
- 接口：`GET /api/events`、`GET /api/events/active`、`POST /api/events`、`POST /api/trips/{tripId}/events/mock`

### 模块 10｜事件—行程影响匹配 🔴
- 10.1 地理距离判定（Haversine / 球面）
- 10.2 时间窗重叠判定
- 10.3 事件关联到具体节点、标记受影响
- 10.4 影响程度分级
- 10.5 生成影响评估记录
- 页面：`/impacts`、`/trips/:id`
- 接口：`POST /api/trips/{tripId}/assess`、`GET /api/trips/{tripId}/impacts`

### 模块 11｜风险评分与预警 🔴
- 11.1 多因子采集（严重度 / 受影响节点数 / 成本暴露 / 剩余缓冲时间）
- 11.2 量纲归一 + 加权，输出 0–100 风险分
- 11.3 风险分级与预警
- 11.4 聚合风险报告 + 因子拆解
- 页面：`/impacts`、`/trips/:id`（风险仪表）
- 接口：`GET /api/trips/{tripId}/risk`

---

## D. 应变恢复与决策

### 模块 12｜自动重规划 🔴
- 12.1 基于影响评估触发重规划
- 12.2 策略一：最低额外成本
- 12.3 策略二：最少延误
- 12.4 策略三：最少改动（最小扰动）
- 12.5 每套方案生成具体节点变更（改期 / 替换 / 删除）+ 增量成本 / 延误 / 改动数 + 文字说明
- 页面：`/plans`
- 接口：`POST /api/trips/{tripId}/replan`、`GET /api/trips/{tripId}/plans`、`GET /api/plans/{planId}`

### 模块 13｜多人协同决策 / 投票 🔴
- 13.1 发起投票（状态机 PROPOSED → VOTING）
- 13.2 成员投票（赞成 / 反对 / 弃权，一人一票 upsert）
- 13.3 实时票数进度
- 13.4 计票（过半判定）
- 13.5 原子应用：采纳方案落地行程、驳回其余方案、幂等保护
- 页面：`/votes`
- 接口：`POST /api/plans/{planId}/start-voting`、`POST /api/plans/{planId}/votes`、`POST /api/plans/{planId}/tally`

### 模块 14｜变更记录 🟢
- 14.1 记录每次采纳的方案变更
- 14.2 额外成本
- 14.3 退款 / 改签截止时间
- 14.4 关联方案与变更列表查询
- 页面：`/trips/:id/changelog`
- 接口：`GET /api/trips/{tripId}/changelogs`

### 模块 15｜消息通知 🟡
- 15.1 WebSocket 实时广播（事件 / 影响 / 新方案 / 投票 / 采纳）
- 15.2 站内信列表，按类型分组
- 15.3 已读 / 未读，单条 / 全部标记已读
- 15.4 订阅鉴权与离线补发
- 页面：`/notifications`
- 接口：`GET /api/notifications`、`POST /api/notifications/{id}/read`、`POST /api/notifications/read-all`；WS `/topic/trips/{tripId}`

- 16.4 结算状态跟踪

### 模块 17｜讨论区 / 评论 🟡
- 17.1 发表评论 / 回复
- 17.2 @成员提及
- 17.3 点赞
- 17.4 关联行程 / 方案的讨论
- 17.5 分页浏览
- 页面：`/discussions`
- 接口：`GET /api/comments`、`POST /api/comments`、`POST /api/comments/{id}/like`

### 模块 18｜数据统计与后台 🟡
- 18.1 出行数量 / 预算分布统计
- 18.2 事件命中率
- 18.3 重规划采纳率
- 18.4 热门目的地图表
- 18.5 用户与事件源管理

## F. 社区与内容闭环

### 模块 19｜攻略社区（浏览 / 分类搜索 / 筛选）🟡
- 19.1 发布攻略（图文 + 关联行程模板）
- 19.2 编辑 / 下架
- 19.3 分类（城市 / 主题 / 天数 / 人均预算 / 季节）
- 19.4 关键词搜索
- 19.5 多条件筛选（城市 / 价格区间 / 天数 / 主题 / 评分）
- 19.6 排序（最新 / 最热 / 评分 / 收藏数）+ 分页
- 19.7 收藏 / 点赞 / 浏览量
- 19.8 评分 / 评价
- 页面：`/guides`、`/guides/:id`
- 接口：`GET /api/guides`、`GET /api/guides/{id}`、`POST /api/guides`、`PUT /api/guides/{id}`、`POST /api/guides/{id}/favorite`、`POST /api/guides/{id}/rate`

### 模块 20｜攻略一键套用（可编辑）+ 定时持续分析 🔴
- 20.1 一键套用：模板复制为目标小组的 Trip + 节点 + 路线
- 20.2 相对时间 → 绝对时间映射（按实际出发日期）
- 20.3 套用后可编辑（改日期 / 删减替换节点 / 调整预算）
- 20.4 套用后成员约束二次校验（日期交集 / 最低预算 / 必访覆盖）+ 冲突提示与自动裁剪
- 20.5 记录来源攻略 `sourceGuideId`
- 20.6 定时持续分析：`@Scheduled` 周期性重跑 assess/risk，仅评估未来节点
- 20.7 风险随剩余时间变化 + 新事件触发式增量分析
- 20.8 分析去重 / 幂等，避免重复生成相同方案
- 页面：`/guides/:id`（套用弹窗）→ `/trips/:id`
- 接口：`POST /api/guides/{id}/apply`（body: groupId、startDate）
