# 外部资源清单（你需要准备 / 调用的东西）

> 说明：标 **【必需】** 的不配项目跑不起来；标 **【可选】** 的不配也能用（有 Mock 或降级）。
> 前端默认 `VITE_USE_MOCKS=true` 时**不需要任何后端与外部资源**即可独立运行演示。下面主要是"接真实后端"时需要的。

---

## 一、基础设施（必需，后端运行依赖）

| 资源 | 用途 | 如何获取 | 配置位置（Spring Boot `application.yml` / 环境变量） |
| --- | --- | --- | --- |
| **MySQL 8** 【必需】 | 主数据库（全部业务表） | 本地安装 / Docker / 云 RDS | `spring.datasource.url=jdbc:mysql://<host>:3306/trip_adaptive`、`spring.datasource.username`、`spring.datasource.password` |
| **Redis** 【必需】 | 会话Token、房间码、热度排行榜、投票锁、缓存、限流、WebSocket 广播 | 本地安装 / Docker / 云 Redis | `spring.data.redis.host`、`spring.data.redis.port`、`spring.data.redis.password` |
| **JDK 17** 【必需】 | 编译运行后端 | Adoptium / OpenJDK | 本机环境 |
| **Maven 3.9+** 【必需】 | 后端构建 | 官网 / 包管理器 | 本机环境（仓库已带 mvnw） |
| **Node.js 18+ / npm** 【必需】 | 前端构建运行 | nodejs.org | 本机环境 |

> Docker 一键起 MySQL + Redis 示例：
> ```bash
> docker run -d --name mysql8 -e MYSQL_ROOT_PASSWORD=devpass -e MYSQL_DATABASE=trip_adaptive -p 3306:3306 mysql:8
> docker run -d --name redis7 -p 6379:6379 redis:7
> ```

## 二、密钥 / 自签配置（必需，自己生成，不用申请）

| 项 | 用途 | 如何获取 | 配置位置 |
| --- | --- | --- | --- |
| **JWT Secret** 【必需】 | 签发/校验登录 Token（模块1鉴权） | 自己生成一段 ≥32 字节随机串，如 `openssl rand -base64 48` | 后端环境变量 `JWT_SECRET`（**勿写进前端/勿提交仓库**） |
| **数据库/Redis 账号密码** 【必需】 | 见上表 | 创建 MySQL 用户并 `GRANT`；Redis 设 `requirepass` | 同上，放环境变量或密钥管理，不进 git |

## 三、第三方 API（按模块，需要去开放平台申请 Key）

| 资源 | 对应模块 | 必需性 | 如何获取（申请入口） | 配置位置 |
| --- | --- | --- | --- | --- |
| **地图开放平台 Key**（百度地图 / 高德地图 二选一） | 模块7 地图路线与导航：距离矩阵 / 路线规划 / polyline | 【可选】不配则降级为 Haversine 直线距离估算 | 百度：`https://lbsyun.baidu.com/`（控制台创建应用拿 **AK**）；高德：`https://lbs.amap.com/`（创建 Key） | `MAP_PROVIDER=baidu|amap`、`MAP_API_KEY=<AK>` |
| **天气 API Key**（和风天气 / 彩云天气 / OpenWeather 任一） | 模块9 事件接入：真实天气（暴雨等） | 【可选】不配则用 Mock 事件 | 和风：`https://dev.qweather.com/`；彩云：`https://caiyunapp.com/`；OpenWeather：`https://openweathermap.org/api` | `WEATHER_API_KEY=<key>`（+ 供应商标识） |
| **交通 / 路况 API**（高德交通态势 等） | 模块9 交通管制/拥堵事件 | 【可选】多数城市无开放公告接口，建议先用 Mock | 高德交通态势 API（同高德控制台）；或各地交通委开放数据 | `TRAFFIC_API_KEY=<key>` |
| **城市活动 / 闭馆公告数据源** | 模块9 大型活动/景点闭馆 | 【可选】无统一公开源，用 Mock / 人工录入 | 无标准 API，可爬公告或人工维护 | — |

> 说明：模块9、模块7 都在文档里明确设计了**降级/ Mock 方案**，所以第三方 API **可以后接**，不阻塞主流程开发。

## 四、增值 / 生产可选服务

| 资源 | 对应模块 | 必需性 | 如何获取 | 配置位置 |

## 六、最小可跑清单（如果只想尽快联调真实后端）
只需准备这 3 样，其余全部可先用 Mock/降级：
1. **MySQL 8**（建库 `trip_adaptive`，导入 `docs/持久化设计.md` 里的 DDL）
2. **Redis**
3. **JWT_SECRET**（自己生成）

地图、天气、交通、OSS、短信、邮件都可以之后按需接入，不影响主流程和演示。
